HTML

  • The head element contains information about the webpage
  • The body element represent the visible content shown to the user
  • Anything inside "< >" is referred to as a tag
  • HTML is the structure of a webpage

CSS

  • Inline CSS: applies styles directly within/to an element
    • Good for quick permanent changes
    • Bad for flexibility as each element would need changed
  • Internal CSS: contains CSS reules for a particular page in the header of the HTML file
    • Only apply to that page
    • Can configure CSS classes and IDs to identify elements and change all of them at once
  • External CSS: External sheet linked within the header
    • Keeps CSS rules seperate, making changes easier

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commits all files in the working branch
  • git commit -m "message here": commits the files to the repository with a comment
  • git pull origin main: pulls the base branch into your feature branch
  • git push origin feature/"": pushes up all of your changes
  • touch "name here" adds a new file

JavaScript

  • A variable is a named container that allows us to sotre data in our code
  • Control flow is the order in which a computer executes code
  • JavaScript works from top to bottom
  • Don't forget to add your ";" after a line of code
  • JavaScript is case sensitive
